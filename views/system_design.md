# Webアプリケーション設計書（system_design.md）

## 1. システム概要
本システムは、任意のデータ（例：駅データ／好きなゲーム一覧／映画データなど）を  
**一覧表示・追加登録できる Web アプリケーション**である。  
サーバーは Node.js + Express、テンプレートエンジンは EJS を使用する。

---

## 2. 対象データ（モデル）
ここでは例として「駅データ」を対象とする。

### **データ項目（スキーマ）**
| 項目名 | 説明 | 型 |
|-------|------|-----|
| id | 一意のID | 数値 |
| code | 駅コード | 文字列 |
| name | 駅名 | 文字列 |
| change | 乗換情報 | 文字列 |
| passengers | 1日の乗車人数 | 数値 |
| distance | 東京駅からの距離 | 数値 |

※ 各自が好きなテーマにする場合  
（例：アニメ、ゲーム、教科書、料理レシピなど）  
→ 項目名は自由に変更してよい。

**例：ゲームデータの場合**
| 項目名 | 説明 |
|-------|------|
| id | 管理ID |
| title | ゲーム名 |
| genre | ジャンル |
| price | 値段 |
| platform | 対応ハード |

## 3. ページ構成・URL一覧

### **① 一覧表示ページ**
- URL：`/keiyo`
- 説明：登録されているデータをすべて表形式で表示
- 使用テンプレート：`db2.ejs`

### **② データ追加フォームページ**
- URL：`/keiyo_add`
- 説明：新しいデータを入力して送信できるフォーム
- 使用テンプレート：`db1.ejs` または `insert.ejs` など

### **③ データ追加後の表示**
- URL：`/keiyo_add`（GET パラメータ付き）
- 説明：データ追加後の画面。  
  表示内容は授業課題により  
  **「入力画面に戻す」または「一覧画面に戻す」** のどちらかを選択。
